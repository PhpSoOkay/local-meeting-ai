# Meeting AI System Tray

Системный трей для Meeting AI — запуск веб-сервера из иконки в трее Ubuntu Linux.

## 📋 Требования

### Системные зависимости

```bash
sudo apt update
sudo apt install python3-gi gir1.2-appindicator3-0.1 gir1.2-gtk-3.0 libnotify-bin
```

### Python зависимости

```bash
# Активируйте виртуальное окружение
source ~/Yandex.Disk/carrier/meeting-ai/.venv/bin/activate

# Установите Pillow для генерации иконок
pip install Pillow
```

## 🚀 Установка

### Автоматическая установка

```bash
cd ~/Yandex.Disk/carrier/meeting-ai
python tray/install.py
```

Скрипт:
- Проверит системные зависимости
- Установит Python зависимости
- Создаст .desktop файл в меню приложений
- Предложит настроить автозапуск

### Настройка автозапуска

```bash
# Включить автозапуск
python tray/install.py --autostart

# Отключить автозапуск
python tray/install.py --no-autostart
```

## 🎯 Использование

### Запуск

1. Найдите **Meeting AI** в меню приложений
2. Кликните для запуска
3. Иконка появится в системном трее (справа вверху)

### Меню в трее

| Пункт меню | Описание |
|------------|----------|
| 🌐 Открыть веб-интерфейс | Открыть браузер на http://localhost:5000 |
| 🔴 Начать запись (встреча) | Начать запись обычной встречи |
| 🔴 Начать Backend Daily | Начать запись ежедневного стендапа |
| ⏹️ Остановить запись | Остановить текущую запись |
| 📋 Открыть консоль логов | Окно с логами сервера |
| ℹ️ Статус | Показать текущий статус |
| 🔄 Перезапустить сервер | Перезапуск Flask-сервера |
| ⏹️ Остановить сервер | Остановить сервер |
| ⚙️ Настройки | Открыть конфигуратор |
| ✕ Выход | Закрыть приложение |

### Иконки в трее

| Состояние | Иконка |
|-----------|--------|
| 🟢 Сервер запущен | Зелёная иконка |
| 🟡 Сервер запускается | Жёлтая иконка |
| 🔴 Сервер остановлен | Красная/серая иконка |
| ⚫ Идёт запись | Красная с индикатором |

### Консоль логов

- Открывается через меню в трее
- Логи обновляются в реальном времени
- Цветовая подсветка уровней (INFO/WARNING/ERROR)
- Автоскролл вниз (можно отключить)
- **Закрытие окна НЕ останавливает сервер**

### Уведомления

Всплывающие уведомления при событиях:
- Сервер запущен
- Запись начата
- Запись остановлена
- Обработка завершена
- Ошибка сервера

## 🐛 Решение проблем

### Иконка не отображается в трее

1. Перезайдите в систему
2. Или перезапустите GNOME Shell: `Alt+F2`, затем `r`

### Ошибка: "требуется GTK 3.0 и AppIndicator3"

```bash
sudo apt install python3-gi gir1.2-appindicator3-0.1
```

### Сервер не запускается

1. Проверьте логи в консоли (меню в трее → Открыть консоль логов)
2. Убедитесь что порт 5000 не занят
3. Проверьте что виртуальное окружение активно

### Уведомления не работают

```bash
# Проверьте что notify-send работает
notify-send "Тест" "Проверка уведомлений"
```

### Ошибка при установке Pillow

```bash
sudo apt install python3-pil
# или
pip install Pillow
```

## 📝 Примечания

- Приложение работает только на Ubuntu/Linux с GNOME
- Требуется PulseAudio/PipeWire для записи звука
- Сервер запускается на `127.0.0.1:5000` по умолчанию
- При выходе из приложения сервер корректно останавливается

## 🎨 Кастомизация

```bash
python tray/app.py
```

```bash
export MEETING_HOST=0.0.0.0  # Доступно из сети
python tray/app.py
```
